# Documentation Gate Checklist

Use this checklist before committing, pushing, opening a PR, or handing off development work.

## Changed Surface

- Did user-facing behavior change?
- Did setup, configuration, testing, deployment, or troubleshooting change?
- Did architecture, data flow, state, permissions, or integrations change?
- Did screenshots, diagrams, maps, or flows become stale?

## Required Updates

- Update text docs where future readers need the change captured.
- Update diagrams, maps, ladder diagrams, screenshots, or other visuals when they clarify the change.
- Keep generated or derived docs in sync when the project expects them.
- Mention explicitly when no documentation update is needed.

## Commit Readiness

- Are documentation changes included in the same commit or PR as the behavior they describe?
- Are visuals current with the implemented behavior?
- Is any larger documentation debt called out separately?
